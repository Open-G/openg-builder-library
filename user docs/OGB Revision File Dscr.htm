This document describes parameters of an OpenG Builder (OGB) Revision File.

 

A Revision File is an optional file that OGB can use to control build versions and save general information on the last build status. A Revision File can treat several Build Files. Information relating to each Build File is stored in a unique section that has name of the Build File.

[BuildFileName]
Version
Build_Number
Build_Date
Status
Warnings
Log_File

 

[BuildFileName]

Version = version string

The Version can accept an arbitrary string to identify a version of a target application. The Version is beyond of the OGB control. However, OGB can use a value of it to assign a version information to a specified control of a VI in a built application as a default value, so the application can obtain the Version string, Build_Number and Build_Date at runtime (see OGB Build File Description, Version Ctrl Name topic for details).

If the Version parameter is not specified in a Revision File, OGB adds the parameter with a value of "0.0.0" and assumes the value as a current version of an application.

Example:

Version = 1.0.0

[BuildFileName]

Build_Number = integer

OGB increments the Build_Number parameter at the beginning of each build. OGB can use a value of it to assign a version information to a specified control of a VI in a built application as a default value, so the application can obtain the Version string, Build_Number and Build_Date at runtime (see OGB Build File Description, Version Ctrl Name topic for details).

If the Build_Number is not specified in a Revision File, it assimes to be zero.

[BuildFileName]

Build_Date = date/time string

OGB assign a date/time string to the Build_Date parameter at the beginning of each build. OGB can use a value of it to assign a version information to a specified control of a VI in a built application as a default value, so the application can obtain the Version string, Build_Number and Build_Date at runtime (see OGB Build File Description, Version Ctrl Name topic for details).

[BuildFileName]

Status = OK/ERROR

Status parameter indicates a status of the last build. It equals "OK" if a build completed with no fatal-error. A value of "ERROR" indicates a fatal-error occurred during a build. Detailed information on the error can be found in a Log_File.

[BuildFileName]

Warnings = integer

Warnings parameter indicates a total number of non-fatal errors and warnings occurred during a build if Status is OK. Detailed information on non-fatal errors and warnings can be found in a Log_File.

[BuildFileName]

Log_File = file path

Log_File parameter indicates a complete path to a log file written during the last build.
