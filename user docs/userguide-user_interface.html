First Impressions

The OpenG Builder UI was designed to look and feel similar to the LabVIEW Application Builder, since most users will already have experience with it. However, when you first look at the OpenG Builder you will notice some immediate differences. The OpenG Builder UI is slightly larger (but, we won't draw any conclusions from that). Also, you will notice that there is no option for choosing a Build Target (for example EXE or DLL). The reason for this, is that building a stand-alone application is very similar to building a reuse library, source distribution, LabVIEW IDE extension, etc. The OpenG Builder allows one to build an executable by optionally converting one of the destination LLBs into an executable (EXE) file. If this destination folder contains a Top-Level VI, then the EXE will auto-launch the Top-Level VI.

General Settings

The General tab allows one to define the Root Paths of the project, the Log File settings, and the Build Version settings. It also allows setting some global parameters, such as Namespace and Password protection.

Root Paths

Pressing the Root Paths... button opens the Root Path Properties dialog.

Log File

Pressing the Log File... button opens the Log File Properties dialog.

Software Version

Pressing the Software Version...button opens the Software Version dialog.

 

Adding Source Files

Add and remove Top-Level and Dynamic VIs to and from the build using the Source Files tab of the OpenG Builder.

 

You can chose whether the path to the Source File is are stored as absolute or Relative to the project's Source Root. Select the Source File from the listbox and toggle the checkbox at the bottom to change this setting.

Notice that the VI Path changes from relative to absolute when the box is checked.

Defining Destinations

A Destination is grouping of files, based upon a specified Source Dir location. Any components of the Source Files' VI hierarchies that are found beneath the Source Dir belong to the Destination group. Each Destination group has unique settings. Some settings can be defined to override the global settings which are defined in the General tab.

Destination Settings

Select a Destination from the Destinations listbox and edit the settings. You can specify paths as abolute or relative by toggling the appropriate checkbox. You can also specify other options for the Destination.

Defining Resource Handling

The OpenG Builder will dynamically discover resource dependencies such as html and chm help files, shared libraries, and external subroutines. You can use the Resources tab to specify the build target location for these files.
